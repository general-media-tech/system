## الأخطاء المُصلَحة في هذه الجلسة

### 1. `gmt-tour.js` — GMTTour غير مُصدَّر
**الخطأ:** `window._gmtTour?.start is not a function`
**السبب:** كلاس `GMTTour` معرَّف داخل الملف لكن لم يُصدَّر لـ `window`
**الإصلاح:** أُضيف `window.GMTTour = GMTTour;` في نهاية الملف

---

### 2. `index-final.html` (POS) — ثلاثة أخطاء

**أ) SyntaxError سطر 2219**
```
Uncaught SyntaxError: Invalid or unexpected token
```
**السبب:** سطر جديد حرفي `\n` داخل string عادي (ليس template literal) في `showLoad()`
**الإصلاح:** دُمج النص في سطر واحد

**ب) `_gmtTour?.start is not a function`**
**السبب:** `gmt-tour.js` يُحمَّل بعد الـ script الذي يستدعي `GMTTour` — الكلاس غير موجود بعد
**الإصلاح:** دالة `gmtInitPosTour()` مؤجَّلة تُعيد المحاولة كل 500ms حتى يتحمَّل `GMTTour`

**ج) رابط المتجر يستخدم `id` بدل الباركود**
**الإصلاح:** الرابط الآن `#product={barcode}` وإذا لم يوجد باركود يرجع لـ `{id}`

---

### 3. `admin-final.html` — await في دوال غير async
**الخطأ:** `Uncaught SyntaxError: await is only valid in async functions`
**السبب:** 7 دوال تستخدم `await` لكنها معرَّفة بـ `function` عادية
**الإصلاح:** كل الدوال التالية أصبحت `async function`:
- `renderCommissions`، `renderInvoices`، `renderOffers`
- `renderCoupons`، `renderDebts`، `renderTransfers`، `renderBranchesTab`

---

### 4. `index__17_.html` (الجرد) — رابط المتجر بالباركود
**قبل:** رابط المتجر يبنى بـ `storeP.id` (ID قاعدة البيانات)
**بعد:** يستخدم `prod.barcode || storeP.barcode || storeP.id`
**التأثير:** زر "عرض في المتجر" يفتح صفحة المنتج مباشرة بنفس الباركود الموجود في الجرد والـ POS

## ما هو موجود ويعمل (لم يُلمَس)

### ربط الأوردر — موجود في POS منذ البداية ✅
- قسم "ربط بنظام الأوردرات" في شاشة الدفع
- زر "🔍 أوردر موجود": بحث بالاسم أو الرقم
- زر "➕ إنشاء أوردر جديد": بيانات الشحن + شركة الشحن
- الربط يحدث تلقائياً عند حفظ الفاتورة

### LocalStorage/IndexedDB في POS ✅
- `gmt_pos_db` (IndexedDB v3) يخزن:
  - `products` — كل المنتجات مع الكاش المحلي
  - `pending_invoices` — فواتير لم تُرفع بعد
  - `branches` — معلومات الفروع
  - `product_images` — صور كـ Blob
  - `meta` — آخر sync وإعدادات
- المزامنة الذكية: تحمّل كاملاً أول مرة، ثم فروق فقط

### ربط المتجر بالباركود ✅
- `bridge-20.html`: ربط الجرد بالمتجر عبر الباركود
- الجرد: يحفظ `store_url` لكل منتج مرتبط
- POS: يقرأ `store_url` من المنتج ويفتح المتجر مباشرة
- **مُحسَّن الآن**: الرابط يستخدم الباركود بدل الـ ID

### نظام المشتريات — النسخ الاحتياطي ✅
- عند حذف فاتورة: تُنزَّل نسخة JSON تلقائياً قبل الحذف
- صيغة: `BACKUP-{رقم الفاتورة}-{التاريخ}.json`
- تتضمن: كل بيانات الفاتورة + المنتجات + الأسعار
